# F-Droid Main Repo With Upstream Reproducible Releases

ShelfDroid will target inclusion in the curated F-Droid main repository rather than a self-hosted F-Droid repo, and it will preserve update continuity for existing GitHub APK users by making each official release tag point at the exact shipped source and by publishing reproducible developer-signed upstream releases that F-Droid can verify against the GitHub APK artifact. ShelfDroid will not treat Google Play installs as part of that continuity target because Google Play App Signing is enabled and only the upload key is locally controlled, so Play-to-F-Droid migration may require reinstall. We accept the extra release discipline, upstream metadata work, and asset provenance audit because broad discovery and seamless migration for the GitHub release line matter more than the fastest possible first listing, and because a post-tag mutated release or a separately signed F-Droid primary artifact would fragment the install base and weaken release provenance.
